Breve introducción#

En este artículo voy a hablar acerca de Linux y dar una visión amplia de este concepto, ya que suele ser un tema un tanto técnico y suele haber confusiones.

De forma general Linux es un sistem operativo, no voy a entrar en detalle en el concepto de este término para no hacer pesado este artículo pero en sí es el software fundamental para usar un dispositivo, ya sea ordenador, tablet móvil o cualquier dispositvo que cuente con una pantalla electrónica (televisores, impresoras con pantalla, microondas con pantalla, máquinas de bebida, etc). Hoy en día casi cualquier dispositvo electrónico cuenta con un sistema operativo integrado, lo que ocurre es que el sistema operativo para una placa de inducción es mucho más sencillo que el de un computador personal o móvil.

Para una placa de inducción solo se necesita configurar el botón (encendido, apagado), botón cambiar potencia (0 a 9) e indicar la posición de la placa, es un sistema operativo muy básico programada a bajo nivel (con funcionamiento directo a la máquina). Sin embargo en un PC hay muchos dispositivos integrados y muchas más tareas que congregar, es un sistema operativo bastante más complejo con distintas capas de funcionamiento.

Por lo general cuando preguntas a la gente ‘¿Qué sistema operativo usas en tu ordenador?’ la mayoría responderá Windows (75% en PC) o macOS (15 % en PC), ya que son los dos SO (Sistemas operativos) más usados a nivel personal debido ya que vienen preinstalados de fábrica, Windows viene con la mayoría de marcas de portátiles y PC de escritorio montados, mientras macOS viene en los ordenadores de Apple. Lo mismo pasa con los móviles la mayoría vienen con versiones de Android y los de Apple (28 % usuarios de móvil) con iOS. Los porcentajes son en computadoras: 77% Windows, 17% macOS, 6% otros; mientras en móviles 72% Google, 27% iOS, 1% otros.

Sin embargo hay otro SO mucho más usado en cómputo total y que la gente no conoce tanto, este es Linux (GNU/Linux concretamente). ¿Y cómo es posible que sea más usado y menos conocido? Pues la respuesta es porque es el SO más usado en servidores, es decir el software predominante en las computadoras que prestan servicio a internet y hay mucho más número de servidores que de dispositivos que reciben servicio. En el caso de servidores, más del 90% son Linux, esto se debe a que los servidores son computadoras sin pantalla que para funcianor solo precisan de la terminal de comandos, los comandos de Linux están más estandarizados, son más compatibles para un entorno web y el software es gratuito de base.

En cambio a nivel de usuario como ordenador personal (PC) no supera el 7% y en el caso de móviles Android es un SO de código abierto basado en Linux pero perteneciente a Google.

Hay mucho más sistemas operativos pero estos tres son los más grandes y potentes. A pesar de ser usados para realizar tareas similares, entre ellos hay grandes diferencias. Windows es un software privativo perteneciente a Microsoft, macOS (OS X) es privativo de la compañía Apple y Linux se basa en la filosofía de código abierto y libre, que no implica gratuito.

Hablemos de Linux#

A Linux se le conoce como un sistema operativo pero estrictamente hablando es el núcleo o kernel del SO que se comunica con el resto programas e interactúa con el hardware. Su nombre completo es GNU/Linux. La combinación del nombre se debe GNU (GNU’s Not Unix! nombre autoreferenciado) proyecto creado por Richard Stallman (estudiante del MIT) en los años 80 con el objetivo de crear un SO de código abierto y libre que remplazase UNIX el SO de la época qu era privativo y de pago, mientras Linux proviene de Linus Torvalds quien contribuyó de manera inicial compartiendo el código de un Kernel de un SO con el proyecto GNU.

El 27 de septiembre de 1983 Richard Stallman anunción el proyecto GNU, proyecto para promover el control y libertad del uso de las computadoras y de la creación de software libre. Ocho años más tarde, el 17 de septiembre de 1991 Linus Torvalds tras 9 meses de trabajo contribuyó con el código escrito para el kernel de el sistema operativo GNU. En ese momento se considera el nacimiento del Linux o concretamente del Kernel de Linux basado en GNU.

Al igual que a Microsoft se le asocia con su fundador Bill Gates, a Apple con Steve Jobs, a Linux se le asocia con dos personas Linus Torvalds (creador del Kernel de Linux) y Richard Stallman (creador del proyecto GNU). Ahora la principal diferencia de Linux frente a Windows y macOS, es su filosofía de trabajo. El sistema operativo de Linux debe cumplir principalmente la regla de código abierto con diferentes licencias de distribución aunque mayoritariamente se ofrece de manera gratuita, a lo que se conoce como FOSS (Free and Open Source Software). Ahora como Linux es solo el Kernel para poder tener el resto de partes que completen el sistema operativo surgen las distribuciones de Linux.

Distros de Linux#

Si quisiéramos montar un sistema operativo de cero personalizable tendríamos que configurar todo el software para hacer funcionar ese hardware, en el caso de Linux para un PC tendrías que configurar el Kernel y el resto del software para ese ordenador. Por eso a lo largo del tiempo se han ido desarrollando lo que se conoce como distribuciones de Linux, sistemas operativos que facilitan el uso e instalación del sistema operativo al usuario. Las distros usan el kernel de Linux añadiendo un instaldor de paquetes, y pueden venir con software preinstalado para su uso directo.

La distribuciones suelen ser de código abierto por tanto ocurre que los desarrollados reutilizan código y las distros van derivando. Cuando surge una nueva distro basada en una original, es decir se crea una rama en el código, se le conoce como distro derivada y si una misma distro tiene distintos entornos para instalar se le conoce como sabores de la distribución. Hoy en día existen cientos de distribuciones de Linux pero la mayoría derivan originalmente de un puñado pequeño.

  • Debian. Debian es de las distros más antigua de 1993. De esta derivan: Ubuntu (ElementaryOS, Linux Mint, KDE Neon, PopOS, Lubuntu, Xubuntu, Zorin), MX-Linux, Parrot, PureOS, Tails, Raspberry, Moodle…
  • Arch Linux creada en 2002, de la cual derivan ArchCraft, ArcoLinux, Artix, Black Arch, Garuda Linux, Hyperbola, Manjaro…
  • Red Hat Linux (Fedora, Mandrake, Slafish OS, Cent OS, Mandriva (Mageia))
  • Gentoo creada en el año 2000.
  • Linux from Scratch, un instalador que permite instalar Linux desde cero, lanzada en el año 1999.
  • Slackware, la primera distro lanzada en el año 1992.
  • Suse (Open Suse, Gecko Linux), lanzada en 1996.
  • Distros independientes: Nix OS (de 2003), KaOS (de KDE), Minimal Linux Live…

Además hay distros de Linux específicas para servidores: Ubuntu Server, Debian, OpenSuse, Fedora Server; o distros especificas como: ZeroShell, ClearLinux (de Intel), OviOS, LTSP. Y para smartwatch (PineTime, Asteroid) o móviles (Android).

Las distros más usadas son basadas en Debian, en Arch y en Red Hat (Fedora). Y la principal diferencia es el instalador de paquetes que usan, el software encargado de facilitar la instalación de aplicaciones.

Distro Caracterśiticas Instalador de paquetes
Debian (Ubuntu, Linuix Mint) Distro estable de por vida, con 4 años de actualizaciones en seguridad apt, dpkg
Red hat Linux (Fedora) Actualizado por versiones, más estable dnf, yum
Arch Linux (ArchCraft, ArcoLinux, Black Arch, Garuda, Manjaro) Actualización continua (posible inestable), repositorio de software de la comunidad AUR pacman

Según el nivel de dificultad:

  • Para principiantes: basadas en Debian con sistema presintalado (Linux Mint, ElementaryOS, Ubuntu, Manjaro)
  • Para usuarios medios: basadas en Arch con entorno preinstalado (Garuda Linux, ArcoLinux) o de Red Hat (Fedora)
  • Para usuarios avanzados: Arch Linux (ejecutando scripts o escribiendo comandos), Slackware o Gentoo

Ubuntu y Linx Mint (basadas en Debian) son muy intuitivas de usar, vienen el sistema operativo configurado con un entorno de escritorio preinstalado y un instalador de aplicaciones. Sin embargo otras distros solo contienen el Kernel con el instalador de paquetes y una terminal de comandos para realizar cualquier cambio, sin interfaz gráfica donde solo funciona el teclado.

En el caso de Arch Linux es para usuarios algo más avanzados, es necesario saber usar la terminal para poder instalar y configurar el sistema operativo a tu gusto, o también copiar y pegar los comandos de la guía de instalación de Arch Linux. La forma clásica de configurar Arch es escribiendo los comandos en la terminal aunque en las nuevas versiones se puede hacer seleccionando un listado de opciones tras ejecutar “archinstall” o ejecutando un script desde un servidor externo o en local. Las tres hacen uso de la terminal de comandos, elemento que le da instrucciones al sistema operativo.

Entre ellos yo estoy usando ahora ArcoLinux que cuenta con multitud de isos con distintos niveles para instalar enfocadas a enseñar al usuario y facilitar la instalación de Arch. Además Arch tiene la ventaja de que funciona siempre en la última versión, eso sí sacrificando un poco de estabilidad y cuenta con una gran comunidad activa. Y otras que recomiendo Fedora o Manjaro.

Instalar software en Linux#

Ahora qué programas necesitamos para montar nuestro propio sistema operativo, pues principalmente:

  1. El núcleo del sistema
  2. Iniciador del sistema: systemd, dinit, runit, upstart, finit.
  3. Controlador de drivers: bluetooth, wifi, impresora, procesador, tarjeta gráfica.
  4. Entorno de escritorio: KDE, GNOME, sway, bspwm, i3
    1. Apps integradas: gestor de inicio, gestor de ventanas, escritorio personalizado (fondos, iconos, tipo de letra)
    2. Apps básicas: terminal, gestor de archivos, navegador, reproductor, editor, utilidades.

Cuando te adentras en Linux descubres que todo es configurable y entiendes más profundamente como funciona un equipo internamente.

Además entiendes que cada distribución precisa de un gestor de paquetes para poder instalar los programas y que la consola (terminal de comandos) cobra mucha más importancia de la que creías. Conocer los comandos de terminal y manejar el uso de ‘scripts’ te convierte en un usuario avanzado, automatizando tareas, instalación de software y control básico de la máquina.

Para instalar apps en Linux antes hay saber si son compatibles con nuestra distribución, en caso afirmativo tenemos varias opciones:

  1. Desde el listado de apps del repositorio de tu distribución
    1. Usar el instalador paquetes propio de cada distro. Ejemplo: en Arch pacman -S firefox, en Debian install apt-get -S firefox.
    2. Con un entorno gráfico que ejecuta los comandos.
  2. Ejecutando un archivo compatible con la distro: Debian (.deb), Red Hat (.rpm)
  3. Con un script en bash y dando permisos de administrador.
  4. Extrayendo los archivos comprimidos e instalándolos siguiendo las instrucciones.
  5. Compilando e instalando desde cero los archivos binarios.

El número de apliaciones nativas en Linux no es tan alto como en Windows sin embargo es más que suficiente para cubrir todas las necesidades. Los problemas vienen con el tema gaming y la edición de contenido, ya que se suelen usar programas privados de Windows o mac.

Los menos benficiados son los que trabajen día a día con Adobe, MS Office, Avid (ProTools), Steinber (Cubase), AutoDesk (AutoCAD, Maya, etc), Apple (Logic Pro), Ableton. Ninguna de estas aplicacione es compatible con Linux sin embargo existen formas de usar aplicaciones nativas de Microsoft Windows en Linux con lo que se conoce como emuladores. El más conocido es WINE, que se usa en Bottles, Lutris, Play on Linux, Proton para facilitar la instalación de WINE.

Personalización de Linux (ricing)#

Linux es muy personalizable tanto que te puedes perder en ese mundo, pero es cuestión de gustos y probar lo que más te guste. A la acción de customizar al detalle la interfaz del sistema se le conoce como ricing y hay comunidades dedicadas a ello en Reedit, en la cual la gente postea sus bonitos entornos de trabajo, la más conocida es r/UnixPorn.

Ventajas y desventajas#

Linux tiene muchas ventajas, la principal es que es de código abierto y libre lo que permite conocer como funciona internamente, la mayoría de las aplicaciones son gratuitas y hay una gran comunidad dispuesta a ayudar. También es más segura en frente a malware debido a que hay menos usuarios por lo que los atacantes se enfocan menos y de base es más robusta frente a ciberataques, además en Linux eres dueño de tus datos no hay trackeo preinstalado por parte de megacorporaciiones como Apple, GOogle o Microsfot.

La principal desventaja es la incompatibilidad con el hardware debido a que su software suele privativo, determinados teclados o ratones con luces la configuración de luces se vuelve inútil, algunas impresoras tampoco funcionan ya que usan drivers privados.

Otra es desventaja es que hay menos aplicaciones compatibles con Linux porque las grandes empresas que desarrolan software privativo de pago no desarrollan para Linux ya que hay menos usuarios y la mayoría usan FOSS por lo que no abría mercado.

Entre las más famosas están Adobe (Adobe Reader, Illustrator, Photoshop), Autodesk (AutoCAD, Maya), Microsoft (MS Office), Serif (Affinity Photo). Además el tema de juegos está enfocado a Windows, ya que Microsoft es dueña de los mayores estudios de videojuegos. Sin embargo todo tiene solución, existen alternativas a la altura de las aplicaciones privativas y formas de poder ejecutar si es necesario desde un emulador de Windows usando Wine (Windows Emulator).

Linux como proceso de aprendizaje#

Una vez somos capaces de configurar nuestro sistema operativo, por ejemplo basado en Arch Linux nos volvemos usuarios más autosuficientes y que entienden mejor el funcionamiento de su sistema. Además con la ventaja de estar usando aplicaciones gratuitas de código abierto, en un entorno seguro y con privacidad, quitándole poder a las macroecosistemas digitales con visión imperialista como son Google, Apple, Microsoft…

Cada vez hay más gente que usa sistemas operativos de código abierto y en concreto basados en Linux, ya que su instalación hoy en día es muy sencilla y hay una gran comunidad dispuesta a resolveer dudas.

Si buscas privacidad aplicaciones de código abierto y libres / aprender y entender mejor el software / eficiencia / minimalismo / configurar tu sistema al detella este es tu sistema operativo.

Conclusiones de Linux#

Linux tiene muchas ventajas es un sistema operativo seguro, estandarizado, de código abierto, gratuito, hay una gran comunidad y el software para este cubre casi todas las necesidades. Además si sabemos usar bien la terminal somos semidioses, podemos hacer de todo.

Como todo hay que adapatarse pero para eso hay probar, una vez te adaptes lo más probable es que te guste y te quedes usando Linux.

Posdata#

Si quieres pasarte a Linux tienes que revisar que el hardware que usas sea compatible con Linux, ya ciertos productos dejarán de funcionar igual si los drivers son únicos de Windows o macOS. Ejemplo ciertas impresoras funcionan solo con Windows, las ajustes de luces RGB precisan de drivers y no todos son compatibles con Linux y así con otros detalles, Por eso es recomendable revisar a la hora de comprar que el hardware sea compatible con Linux.

Aunque es cierto que hoy en día cada vez las empresas se abren más a liberar código de aplicaciones antiguas y la comunidad es capaz de crear aplicaciones que solucionan esos problemas.

Notas#

Distros: acrónimo de distribuciones. Distros derivadas: son distros basadas en una original pero independientes. Sabores de distro: distro con cambios pequeños en la apariencia. Especias de distros: son complementos para una misma distribución. SO: Sistema Operativo.

FOSS: Free and Open Source Software
