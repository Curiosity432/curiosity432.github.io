Brief introduction#

In this article I am going to talk about Linux and give a broad view of this concept, since it is usually a somewhat technical subject and there is often confusion.

In general, Linux is an operating system, I am not going to go into detail about the concept of this term so as not to make this article heavy, but in itself it is the fundamental software to use a device, be it a computer, a mobile tablet or any other device that counts. with an electronic screen (televisions, printers with screens, microwaves with screens, drink machines, etc.). Today almost any electronic device has an integrated operating system, what happens is that the operating system for an induction hob is much simpler than that of a personal or mobile computer.

For an induction hob you only need to configure the button (on, off), change power button (0 to 9) and indicate the position of the hob, it is a very basic operating system programmed at a low level (with direct operation to the machine ). However, in a PC there are many integrated devices and many more tasks to bring together, it is a much more complex operating system with different layers of operation.

Usually when you ask people ‘What operating system do you use on your computer?’ most will answer Windows (75% on PCs) or macOS (15% on PCs), since they are the two most used OS (Operating Systems) on a personal level because they come pre-installed from the factory, Windows comes with most brands of assembled laptops and desktop PCs, while macOS comes on Apple computers. The same happens with mobile phones, most of them come with versions of Android and those of Apple (28% mobile users) with iOS. The percentages are in computers: 77% Windows, 17% macOS, 6% other; while on mobile phones 72% Google, 27% iOS, 1% others.

However, there is another OS much more used in total computing and that people do not know so much, this is Linux (GNU/Linux specifically). And how is it possible that it is more used and less known? Well, the answer is because it is the most used OS in servers, that is, the predominant software in computers that provide Internet service and there are much more servers than devices that receive service. In the case of servers, more than 90% are Linux, this is because the servers are computers without a screen that only require the command terminal to function, Linux commands are more standardized, they are more compatible for a web environment and the software is free of charge.

On the other hand, at the user level as a personal computer (PC) it does not exceed 7% and in the case of Android mobiles it is an open source OS based on Linux but belonging to Google.

There are many more operating systems but these three are the biggest and most powerful. Despite being used to perform similar tasks, there are big differences between them. Windows is proprietary software belonging to Microsoft, macOS (OS X) is proprietary to Apple, and Linux is based on the philosophy of open source and free, which does not mean free.

Let’s talk Linux#

Linux is known as an operating system but strictly speaking it is the kernel of the OS that communicates with the rest of the programs and interacts with the hardware. Its full name is GNU/Linux. The combination of the name is due to GNU (GNU’s Not Unix! self-referenced name) project created by Richard Stallman (MIT student) in the 80s with the aim of creating a free and open source OS that would replace UNIX the OS of the time that it was proprietary and paid, while Linux comes from Linus Torvalds who initially contributed by sharing the code of an OS Kernel with the GNU project.

On September 27, 1983, Richard Stallman announced the GNU project, a project to promote control and freedom in the use of computers and the creation of free software. Eight years later, on September 17, 1991, after 9 months of work, Linus Torvalds contributed the code written for the kernel of the GNU operating system. At that time, the birth of Linux or specifically the Linux Kernel based on GNU is considered.

Just as Microsoft is associated with its founder Bill Gates, Apple with Steve Jobs, Linux is associated with two people Linus Torvalds (creator of the Linux Kernel) and Richard Stallman (creator of the GNU project). Now the main difference between Linux and Windows and macOS is its work philosophy. The Linux operating system must mainly comply with the open source rule with different distribution licenses, although it is mostly offered free of charge, known as FOSS (Free and Open Source Software). Now, as Linux is only the Kernel, in order to have the rest of the parts that complete the operating system, Linux distributions arise.

Linux Distros#

If we wanted to mount an operating system of zero customizable we would have to configure all the software to make that hardware work, in the case of Linux for a PC you would have to configure the Kernel and the rest of the software for that computer. For this reason, over time, what are known as Linux distributions have been developed, operating systems that facilitate the use and installation of the operating system for the user. Distros use the Linux kernel by adding a package installer, and may come with software pre-installed for direct use.

The distributions are usually open source, so it happens that the developers reuse code and the distros are derived. When a new distro appears based on an original one, that is, a branch is created in the code, it is known as a derived distro and if the same distro has different environments to install, it is known as flavors of the distribution. Today there are hundreds of Linux distributions but most are originally derived from a small handful.

-Debian. Debian is one of the oldest distros from 1993. Ubuntu (ElementaryOS, Linux Mint, KDE Neon, PopOS, Lubuntu, Xubuntu, Zorin), MX-Linux, Parrot, PureOS, Tails, Raspberry, Moodle…

  • Arch Linux created in 2002, from which derive ArchCraft, ArcoLinux, Artix, Black Arch, Garuda Linux, Hyperbola, Manjaro…
  • Red Hat Linux (Fedora, Mandrake, Slafish OS, Cent OS, Mandriva (Mageia))
  • Gentoo created in the year 2000.
  • Linux from Scratch, an installer that allows you to install Linux from scratch, released in 1999.
  • Slackware, the first distro released in 1992.
  • Suse (Open Suse, Gecko Linux), released in 1996.
  • Independent distros: Nix OS (from 2003), KaOS (from KDE), Minimal Linux Live…

In addition there are specific Linux distros for servers: Ubuntu Server, Debian, OpenSuse, Fedora Server; or specific distros like: ZeroShell, ClearLinux (from Intel), OviOS, LTSP. And for smartwatch (PineTime, Asteroid) or mobile (Android).

The most used distros are based on Debian, Arch and Red Hat (Fedora). And the main difference is the package installer they use, the software in charge of facilitating the installation of applications.

District Features Package Installer
Debian (Ubuntu, Linux Mint) Stable distro for life, with 4 years of security updates apt,dpkg
Red Hat Linux (Fedora) Updated by versions, more stable dnf, yum
Arch Linux (ArchCraft, ArcoLinux, Black Arch, Garuda, Manjaro) Continuous update (possibly unstable), AUR Community Software Repository pac-man

Depending on the level of difficulty:

  • For beginners: Debian-based with pre-installed system (Linux Mint, ElementaryOS, Ubuntu)
  • For average users: based on Arch with pre-installed environment (Garuda Linux, ArcoLinux) or Red Hat (Fedora)
  • For advanced users: Arch Linux (running scripts or typing commands), Slackware or Gentoo

Ubuntu and Linx Mint (based on Debian) are very intuitive to use, they come with the operating system configured with a pre-installed desktop environment and an application installer. However, other distros only contain the Kernel with the package installer and a command terminal to make any changes, without a graphical interface where only the keyboard works.

In the case of Arch Linux, it is for somewhat more advanced users, it is necessary to know how to use the terminal to be able to install and configure the operating system to your liking, or also copy and paste the commands from the [Arch Linux installation guide](https ://wiki.archlinux.org/title/Installation_guide). The classic way to configure Arch is by typing the commands in the terminal, although in the new versions it can be done by selecting a list of options after executing “archinstall” or by executing a script from an external or local server. All three make use of the command terminal, an element that gives instructions to the operating system.

Among them, I am now using ArcoLinux, which has a multitude of isos with different levels to install, focused on teaching the user and facilitating the installation of Arch. In addition, Arch has the advantage that it always works in the latest version, albeit sacrificing a bit of stability and has a large active community. And others that I recommend Fedora or Manjaro.

Install software on Linux#

Now what programs do we need to mount our own operating system, mainly:

  1. The core of the system
  2. System launcher: systemd, dinit, runit, upstart, finit.
  3. Driver controller: bluetooth, wifi, printer, processor, graphics card.
  4. Desktop environment: KDE, GNOME, sway, bspwm, i3
  5. Integrated Apps: Start Manager, Window Manager, Custom Desktop (Backgrounds, Icons, Fonts)
  6. Basic apps: terminal, file manager, browser, player, editor, utilities.

When you get into Linux you discover that everything is configurable and you understand more deeply how a computer works internally.

You also understand that each distribution needs a package manager to be able to install the programs and that the console (command terminal) is much more important than you thought. Knowing the terminal commands and handling the use of ‘scripts’ makes you an advanced user, automating tasks, software installation and basic control of the machine.

To install apps on Linux, you must first know if they are compatible with our distribution, if so, we have several options:

  1. From the list of apps in the repository of your distribution
  2. Use the package installer of each distro. Example: on Arch pacman -S firefox, on Debian install apt-get -S firefox.
  3. With a graphical environment that executes the commands.
  4. Running a distro compatible file: Debian (.deb), Red Hat (.rpm)
  5. With a bash script and giving administrator permissions.
  6. Extracting the compressed files and installing them following the instructions.
  7. Compiling and installing the binaries from scratch.

The number of native applications in Linux is not as high as in Windows, however it is more than enough to cover all needs. The problems come with the gaming theme and content editing, since private Windows or Mac programs are usually used.

The least benefited are those who work every day with Adobe, MS Office, Avid (ProTools), Steinber (Cubase), AutoDesk (AutoCAD, Maya, etc), Apple (Logic Pro), Ableton. None of these applications are compatible with Linux however there are ways to use native Microsoft Windows applications on Linux with what are known as emulators. The best known is WINE, which is used by Bottles, Lutris, Play on Linux, Proton to make installing WINE easier.

Linux customization (ricing)#

Linux is very customizable so much that you can get lost in that world, but it is a matter of taste and try what you like best. The action of customizing the system interface in detail is known as ricing and there are communities dedicated to it in Reedit, in which people post their beautiful work environments, the best known is [r/UnixPorn](https:/ /www.reddit.com/r/unixporn).

Advantages and disadvantages#

Linux has many advantages, the main one is that it is open source and free which allows you to know how it works internally, most of the applications are free and there is a large community willing to help. It is also safer against malware because there are fewer users, so attackers focus less and it is more robust against cyberattacks, also in Linux you own your data there is no pre-installed tracking by mega-corporations like Apple , GOogle or Microsoft.

The main disadvantage is the incompatibility with the hardware due to the fact that its software is usually proprietary, certain keyboards or mice with lights, the light configuration becomes useless, some printers do not work either since they use private drivers.

Another disadvantage is that there are fewer applications compatible with Linux because the large companies that develop proprietary paid software do not develop for Linux since there are fewer users and most use FOSS, so there was no market.

Among the most famous are Adobe (Adobe Reader, Illustrator, Photoshop), Autodesk (AutoCAD, Maya), Microsoft (MS Office), Serif (Affinity Photo). In addition, the theme of games is focused on Windows, since Microsoft owns the largest video game studios. However, everything has a solution, there are alternatives to the level of proprietary applications and ways to run if necessary from a Windows emulator using Wine (Windows Emulator).

Linux as a learning process#

Once we are able to configure our operating system, for example based on Arch Linux, we become more self-sufficient users who better understand how their system works. Also with the advantage of using free open source applications, in a safe and private environment, taking power away from digital macroecosystems with an imperialist vision such as Google, Apple, Microsoft…

There are more and more people who use open source operating systems and specifically based on Linux, since its installation today is very simple and there is a large community willing to answer questions.

If you are looking for privacy, open source and free applications / learn and better understand software / efficiency / minimalism / configure your system in detail, this is your operating system.

Linux Conclusions#

Linux has many advantages, it is a safe, standardized, open source, free operating system, there is a large community and the software for it covers almost all needs. Also, if we know how to use the terminal well, we are demigods, we can do anything.

Like everything you have to adapt but for that you have to try, once you adapt you will most likely like it and stay using Linux.

Postscript#

If you want to switch to Linux you have to check that the hardware you use is compatible with Linux, and certain products will stop working the same if the drivers are unique to Windows or macOS. For example, certain printers work only with Windows, RGB light settings require drivers and not all of them are compatible with Linux and so on, so it is advisable to check when buying that the hardware is compatible with Linux.

Although it is true that today companies are increasingly open to releasing code from old applications and the community is capable of creating applications that solve these problems.
